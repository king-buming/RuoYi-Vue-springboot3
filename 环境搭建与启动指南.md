# RuoYi-Vue 环境搭建与启动指南

---

## 一、所需软件

| 软件 | 版本要求 | 用途 | 默认端口 |
|------|----------|------|---------|
| JDK | 17+ | Java 开发与运行 | - |
| MySQL | 8.x | 关系型数据库 | 3306 |
| Redis | 任意版本 | 缓存/Token/验证码存储 | 6379 |
| Maven | 3.6+ | 后端依赖管理与构建 | - |
| Node.js + npm | >= 8.9 | 前端构建与开发 | - |
| IDEA | 任意版本 | Java IDE（你已安装） | - |

---

## 二、准备工作

### 1. 本机环境确认

你的 JDK 17 和 IDEA 已安装，还需确认以下软件是否已安装：

```bash
# 检查 Java 版本
java -version

# 检查 MySQL
mysql --version

# 检查 Redis
redis-cli --version

# 检查 Maven
mvn -version

# 检查 Node.js
node -v
npm -v
```

如有未安装的，先完成安装再继续。

### 2. 创建数据库并导入数据

启动 MySQL 后，执行以下操作：

#### 创建数据库

```sql
CREATE DATABASE `ry-vue` DEFAULT CHARACTER SET utf8 COLLATE utf8_general_ci;
```

#### 导入数据

使用命令行或数据库管理工具（Navicat、DBeaver 等）导入 `sql/` 目录下的两个文件：

```
sql/
├── ry_20260417.sql   ← 先导入（系统核心表：用户/角色/菜单/部门等）
└── quartz.sql        ← 后导入（定时任务相关表）
```

命令行导入方式（在项目根目录执行）：

```bash
mysql -u root -p ry-vue < sql/ry_20260417.sql
mysql -u root -p ry-vue < sql/quartz.sql
```

### 3. 修改数据库连接配置

编辑 `ruoyi-admin/src/main/resources/application-druid.yml`，修改第 10-11 行的数据库用户名和密码：

```yaml
master:
    url: jdbc:mysql://localhost:3306/ry-vue?...
    username: root          # ← 改成你的 MySQL 用户名
    password: password      # ← 改成你的 MySQL 密码
```

### 4. 修改 Redis 连接配置（如果需要）

编辑 `ruoyi-admin/src/main/resources/application.yml`，如果你的 Redis 设置了密码：

```yaml
data:
    redis:
        host: localhost
        port: 6379
        password:           # ← 如果 Redis 有密码，填在这里
```

如果 Redis 没有设置密码，可以不动这个配置。

### 5. 配置 Maven（建议）

为保证依赖下载速度，建议在 Maven 的 `settings.xml` 中配置阿里云镜像：

```xml
<mirror>
    <id>aliyun</id>
    <name>aliyun maven</name>
    <mirrorOf>central</mirrorOf>
    <url>https://maven.aliyun.com/repository/public</url>
</mirror>
```

在 IDEA 中 `File → Settings → Build → Build Tools → Maven` 中确认 Maven 路径和 `settings.xml` 路径。

## 三、启动步骤

按以下顺序依次启动：

### 第一步：启动 MySQL 和 Redis

确保两个服务都在运行中。Windows 下可在"服务"（`services.msc`）中确认或通过命令行启动。

### 第二步：启动后端

#### 方式一：IDEA 启动（推荐）

1. 用 IDEA 打开项目根目录（`pom.xml` 所在目录，即 `RuoYi-Vue-master`）
2. IDEA 会自动识别为 Maven 项目，等待依赖下载完成
3. 找到启动类：`ruoyi-admin/src/main/java/com/ruoyi/RuoYiApplication.java`
4. 右键 → **Run 'RuoYiApplication'**

#### 方式二：命令行启动

```bash
cd RuoYi-Vue-master
mvn clean install -DskipTests
cd ruoyi-admin
mvn spring-boot:run
```

#### 启动成功标志

控制台输出若依的 ASCII Art 横幅：

后端默认运行在 **http://localhost:8080**。

### 第三步：启动前端

```bash
cd ruoyi-ui

# 安装依赖（仅首次需要）
npm install --registry=https://registry.npmmirror.com

# 启动开发服务器
npm run dev
```

前端默认运行在 **http://localhost:80**。

## 四、验证

1. 打开浏览器访问 `http://localhost`
2. 登录页应能看到验证码（验证码正常显示 = Redis 工作正常）
3. 使用管理员账号登录：`admin` / `admin123`
4. 登录成功后进入系统首页仪表盘

### 常用访问地址

| 地址 | 说明 |
|------|------|
| `http://localhost` | 前端登录页面 |
| `http://localhost:8080` | 后端服务 |
| `http://localhost:8080/swagger-ui.html` | API 接口文档 |
| `http://localhost:8080/druid/` | Druid 数据库监控面板 |

## 五、常见问题排查

| 现象 | 可能原因 | 解决方法 |
|------|----------|---------|
| 后端启动报数据库连接失败 | MySQL 未启动或密码错误 | 检查 MySQL 服务状态，确认 `application-druid.yml` 配置 |
| 验证码不显示 | Redis 未启动 | 启动 Redis 服务 |
| 登录页不显示 | 前端未启动 | 执行 `cd ruoyi-ui && npm run dev` |
| `npm install` 报错 | Node 版本过低 | 升级到 Node 16+ |
| 登录报 401 | Token 相关错误 | 检查 Redis 是否正常、JWT 配置是否正确 |
| 页面 404 | 前端代理未生效 | 检查 `ruoyi-ui/vue.config.js` 中的代理配置 |

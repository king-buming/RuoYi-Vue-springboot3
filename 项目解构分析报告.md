## 二、项目目录结构说明

### 2.1 Maven 模块结构（后端 6 模块 + 前端 1 模块）

```
RuoYi-Vue-master/
├── ruoyi-admin          # [主入口模块] — Spring Boot 启动、Controller、配置文件
├── ruoyi-common         # [公共工具模块] — 通用注解、工具类、基础异常、XSS过滤器
├── ruoyi-framework      # [框架核心模块] — 安全认证、AOP、数据源、拦截器、Web服务层
├── ruoyi-system         # [系统业务模块] — 系统实体(Domain/Mapper/Service)、业务逻辑
├── ruoyi-quartz         # [定时任务模块] — Quartz 调度任务的管理与执行
├── ruoyi-generator      # [代码生成模块] — 代码生成器(Controller/Mapper/模板)
├── ruoyi-ui             # [前端模块] — Vue 2.x 前端SPA应用
├── sql/                 # [数据库脚本] — 初始化 SQL（含 Quartz 及主系统数据）
├── bin/                 # [运维脚本] — 启动/停止脚本
├── doc/                 # [文档]
├── pom.xml              # Maven 父 POM，依赖声明 + 模块聚合
├── ry.bat / ry.sh       # 启动快捷脚本
└── LICENSE / README.md
```

## 四、架构特点总结

### 请求/响应规范

- **统一响应体**：`AjaxResult` 包含 `code`、`msg`、`data` 三个字段
- **分页响应体**：`TableDataInfo` 包含 `code`、`msg`、`rows`、`total`
- **认证方式**：JWT Token，通过 `Authorization` 请求头传递
- **权限控制**：Spring Security + `@PreAuthorize("@ss.hasPermi('...')")` 注解，结合 `system:*:*` 通配符权限字符串
- **操作日志**：通过自定义 `@Log` 注解 + AOP 切面自动记录
- **防重提交**：`@RepeatSubmit` 注解 + Redis 缓存拦截

### 前端调用模式

- 所有 API 调用统一经过 `src/utils/request.js` 封装的 Axios 实例
- 请求自动携带 Token（除 login、captchaImage、register 外）
- 响应拦截器统一处理错误码（401 跳转登录页）
- 前端 API 模块与服务端 Controller 路径一一对应

### 数据源

- 主数据库：`ry-vue`（MySQL）
- 支持主从读写分离（从库默认关闭）
- 缓存：Redis（用于 Token 存储、验证码、配置缓存、字典缓存、防重提交、限流）

### 潜在关注点

1. JWT 密钥 `abcdefghijklmnopqrstuvwxyz` 和 Druid 监控面板密码 `123456` 均为默认值，**生产环境必须修改**
2. Redis 默认无密码，生产应设置密码
3. `application.yml` 中 CORS 未显式配置，若前端独立部署需增加跨域配置
4. Springdoc 仅扫描了 `tool` 测试模块的 API，其余 Controller 的接口文档未自动生成

## 一、数据库变更

### 1.1 新增业务表（7 张 tb_worker 系列 + 1 张微信绑定表）

| 表名 | 说明 | 主要字段 |
|------|------|---------|
| `tb_worker` | 人员基础档案（核心） | id, worker_name, phone, id_card, gender, unit_type(1管网/2第三方/3施工方), status(0在场/1离场/2禁用), face_status, audit_status, del_flag |
| `tb_worker_role` | 角色规则表 | role_code, role_name, unit_type, fixed_site_flag, need_sign_in/out, need_hourly_check, hourly_interval, need_cert, cert_type, is_self_checkin(0被动/1自主) |
| `tb_worker_role_rel` | 人员-角色多对多 | worker_id, role_id（UNIQUE约束） |
| `tb_worker_cert` | 资质证件 | cert_type, cert_no, issue_date/expire_date, cert_img, audit_status |
| `tb_worker_face` | 人脸信息 | worker_id, face_img_url, face_feature（TEXT，AI预留）, collect_time |
| `tb_worker_audit` | 审核记录 | biz_type(worker/cert), biz_id, worker_id, audit_status, audit_opinion, auditor, audit_time |
| `tb_worker_checkin` | 打卡记录 | check_type(1签到/2签退/3点到), check_time, check_method, site_id, photo_url, latitude, longitude, ai_result(预留), helmet_flag, vest_flag |
| `tb_worker_wechat_bind` | 微信绑定 | worker_id, openid(UNIQUE), unionid, bind_time, status |

**文件位置**：`sql/updates/20260601_0900_worker_module.sql`

### 2.2 手机端 API（/app/* 命名空间，4 个 Controller）

| Controller | 接口 | 说明 |
|-----------|------|------|
| AppAuthController | POST /app/auth/login | 手机号+身份证后6位登录，返回 HMAC Token |
| | GET /app/auth/me | 获取当前人员信息（含角色名称、自主打卡标记） |
| AppCheckinController | GET /app/checkin/today | 今日签到签退状态 |
| | POST /app/checkin/signIn | 签到（含拍照上传 + 规则校验） |
| | POST /app/checkin/signOut | 签退（含时间防作弊 + 规则校验） |
| | GET /app/checkin/list | 我的打卡记录 |
| | GET /app/checkin/project-status | 同单位人员打卡状态 |
| AppWorkerController | GET /app/worker/profile | 个人资料 |
| | GET/POST /app/worker/certs | 资质证件上传/查看 |
| | GET/POST /app/worker/face | 人脸照片上传/查看（上传自动回写 face_status） |
| AppCommonController | GET /app/common/dicts | 字典数据（证件类型等，动态加载） |

### 2.3 安全机制

| 组件 | 说明 |
|------|------|
| AppTokenUtil | HMAC-SHA256 签发/验证，7 天有效期 |
| AppTokenFilter | 统一拦截 /app/*，无 Token 返回 401（login/dicts 匿名放行） |
| SecurityConfig | /app/** 和 /common/upload 加入 permitAll |

### 2.4 核心业务规则

| 规则 | 实现位置 | 说明 |
|------|---------|------|
| 审核回写 | TbWorkerAuditServiceImpl.syncStatus() | @Transactional，新增/修改审核记录时同步更新主表 audit_status |
| 人员审核通过 → 同步所有证件 | TbWorkerAuditServiceImpl.insertTbWorkerAudit() | 资质齐全→人员+全部证件通过；缺证件→警告+保持待审核 |
| 角色资质弱约束 | TbWorkerServiceImpl.validateRoleRequirements() | 缺证件时黄色警告+退回待审核，不阻断分配 |
| 逻辑删除 | TbWorkerController.remove() | del_flag='2'+status='2'，关联数据全保留 |
| 归档拦截 | ITbWorkerService.isWorkerActive() | del_flag='0' 且 status!='2'，所有写接口前置校验 |
| 身份证唯一 | TbWorkerServiceImpl.checkIdCardUnique() | 新增/修改时后端查重 |
| 打卡防重复 | AppCheckinController.doCheck() | 同一天签到/签退只能各一次 |
| 签退防作弊 | AppCheckinController.doCheck() | 签退时间必须晚于签到时间 |
| 自主打卡权限 | AppAuthController.login() | 施工人员(is_self_checkin='0')只能登记信息，不能打卡 |
| 证件过期检测 | CertExpiryJob | Quartz 每天 02:00 扫描，过期→ audit_status='3' |

## 三、前端开发

### 3.1 后台管理页面（6 个 Vue 页面）

**位置**：`ruoyi-ui/src/views/worker/`

| 页面 | 文件 | 关键功能 |
|------|------|---------|
| 人员档案 | worker/index.vue | 角色多选分配、身份证自动推断性别、单位类型字典、关联数据聚合视图(身份证/资质/人脸/审核) |
| 角色规则 | role/index.vue | 联动字段(点到→间隔、资质→类型)、自主打卡开关 |
| 资质证件 | cert/index.vue | 人员下拉搜索+姓名显示、图片上传、已归档标记 |
| 人脸信息 | face/index.vue | 照片上传+预览、采集时间 |
| 审核记录 | audit/index.vue | 人员审核弹窗展示人脸+身份证+资质缩略图、缺资质警告 |
| 打卡记录 | checkin/index.vue | 人员下拉、时间范围 params 区间查询、导出带时间参数 |

**API 模块**：`ruoyi-ui/src/api/worker/`（6 个 JS 文件）

### 3.2 手机端 H5（mobile.html，纯单文件）

**位置**：`ruoyi-ui/public/mobile.html`（Vue dev server 托管）

| Tab | 功能 |
|-----|------|
| 登录 | 手机号+密码（身份证后6位，密文输入），Token 7天免登 |
| 打卡 | 实时时钟、签到签退按钮、拍照上传、今日状态显示 |
| 工作台 | 方块卡片布局：打卡记录、同项目人员状态、班前喊话(预留)、统一签退(预留)、消息通知(预留) |
| 记录 | 历史打卡列表（签到绿/签退红/点到橙）+ 时间筛选 |
| 我的 | 个人资料卡片(含角色)、人脸状态、身份证列表(人像面/国徽面)、资质列表(带缩略图) |
| 上传身份证 | 身份证号+到期日+人像面+国徽面，两条记录 |
| 上传资质 | 动态类型下拉+多图上传(逗号拼接URL) |
| 人脸采集 | 正脸照上传，自动标记 face_status='1' |

**施工人员特殊处理**：
- is_self_checkin='0' → 登录后仅显示「我的」Tab
- 可上传身份证、资质、人脸（AI 识别基础数据）
- 不显示打卡/工作台

## 一、合并策略

### 1.1 核心原则

两个项目使用**完全独立的包路径、数据库表前缀、Controller 路径和菜单体系**，天然解耦，无需修改任何一方的业务代码即可共存：

| 维度 | 主项目（homework） | ruoyiV1（worker） | 冲突 |
|------|-------------------|-------------------|:--:|
| Java 包 | `com.ruoyi.system.*` | `com.ruoyi.worker.*` | 无 |
| 数据库表 | `hw_plan`, `hw_worker`, `hw_attendance` | `tb_worker`, `tb_worker_role`, `tb_worker_cert`, `tb_worker_face`, `tb_worker_audit`, `tb_worker_checkin`, `tb_worker_role_rel` | 无 |
| Controller 路径 | `/homework/**`, `/wechat/**` | `/worker/**`, `/app/**` | 无 |
| 侧边栏菜单 | 「作业管理」— 3 个子菜单 | 「人员管理」— 6 个子菜单 | 无 |
| MyBatis XML 目录 | `mapper/system/` | `mapper/worker/` | 无 |
| 前端 API 目录 | `src/api/homework/` | `src/api/worker/` | 无 |
| 前端页面目录 | `src/views/homework/` | `src/views/worker/` | 无 |

### 1.2 注意事项

虽然表前缀和包路径不同，但两个项目在**业务语义上有重叠**：
- 主项目的 `hw_worker`（施工人员表，9角色，简单字段）和 ruoyiV1 的 `tb_worker`（人员基础档案 + 6 张关联表）都管理"人员"
- 主项目的 `hw_attendance`（打卡记录）和 ruoyiV1 的 `tb_worker_checkin`（打卡记录）都记录打卡
- 主项目的 `/wechat/attendance/*` 和 ruoyiV1 的 `/app/checkin/*` 都提供移动端打卡

这些是**两个开发人员独立实现的同一领域功能**，表结构、字段设计、接口路径均不同。当前阶段保持两者并存，后续可根据业务需求统一。

### 未合并的共享文件（及原因）

ruoyiV1 修改了大量 RuoYi 框架和前端基础设施文件，经评估后**不合并**以下类别：

| 类别 | 文件数 | 不合并原因 |
|------|:---:|------|
| 公共工具类增强 | ~10 | `R.java`、`PageUtils.java` 等新增方法，Worker 代码不依赖 |
| 前端框架文件 | ~35 | `permission.js`、`request.js`、`router/index.js` 等改动较大，强行覆盖可能破坏 homework 模块 |
| Vue 组件/布局/样式 | ~20 | Element UI 主题、布局组件微调，非 Worker 模块运行必须 |
| 系统管理页面 | ~8 | user/dept/post/menu 等页面增强，非必须 |
| Velocity 模板 | 12 | 代码生成器模板全量更新，非必须 |
| 监控模块 | ~6 | job/logininfor/operlog 页面调整，非必须 |

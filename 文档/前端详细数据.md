# 若依（RuoYi）前端详细解析文档

> 基于 RuoYi-Vue v3.9.2 SpringBoot3 版本  
> 技术栈：Vue 2.6.12 + Vuex 3.6.0 + Vue Router 3.4.9 + Element UI 2.15.14 + Axios 0.30.3

---

## 一、项目总览

### 1.1 整体架构图

```
浏览器打开 http://localhost
        │
        ▼
   public/index.html          ← 入口 HTML，包含加载动画
        │
        ▼
   src/main.js                ← 应用入口，初始化 Vue 实例
        │
        ├── App.vue           ← 根组件，只放 <router-view> 和主题选择器
        │
        ├── permission.js     ← 路由守卫（核心！控制登录/权限/动态路由）
        │
        ├── router/           ← 路由配置（静态路由 + 动态路由）
        │   └── index.js
        │
        ├── store/            ← Vuex 状态管理（7个模块）
        │   ├── modules/user.js       ← 用户信息、Token、登录/退出
        │   ├── modules/permission.js ← 动态路由生成（核心！）
        │   ├── modules/app.js        ← 侧边栏状态、设备类型
        │   ├── modules/settings.js   ← 主题、布局配置
        │   ├── modules/tagsView.js   ← 页签管理
        │   ├── modules/dict.js       ← 数据字典缓存
        │   └── modules/lock.js       ← 屏幕锁定
        │
        ├── api/              ← 后端接口调用（按模块分目录）
        │
        ├── utils/request.js  ← Axios 封装（拦截器、Token注入、防重提交）
        │
        ├── layout/           ← 页面布局框架（侧边栏、导航栏、页签栏、主内容区）
        │
        ├── views/            ← 业务页面（按模块分目录）
        │
        ├── components/       ← 全局通用组件
        │
        ├── directive/        ← 自定义指令（权限、拖拽、剪贴板）
        │
        ├── plugins/          ← Vue 插件（$tab, $auth, $cache, $modal, $download）
        │
        └── assets/           ← 静态资源（图标、样式、图片）
```

### 1.2 前端三大核心闭环

| 闭环 | 流程 | 说明 |
|------|------|------|
| **登录闭环** | 输入账号密码 → 调用 `/login` → 获取 Token 存 Cookie → 调用 `/getInfo` 获取用户信息（角色+权限）→ 调用 `/getRouters` 获取菜单路由 → 动态添加路由 → 跳转首页 | permission.js 路由守卫驱动 |
| **请求闭环** | 页面调用 api/xxx.js → request.js 拦截器自动注入 `Authorization: Bearer <token>` → 发送到后端 → 响应拦截器处理 401/500 等状态码 → 返回数据 | request.js Axios 实例驱动 |
| **渲染闭环** | 路由变化 → Layout 布局渲染 → 侧边栏菜单（sidebarRouters）→ 主体区 `<router-view>` → 业务页面 → 页面内调用 API 获取数据 → Element UI 表格/表单展示 | Vue Router + Layout 驱动 |

### 3.4 `permission.js` — 路由守卫（核心文件）

```
位置：src/permission.js
```

这是整个前端权限控制的核心。使用 `router.beforeEach` 拦截每一次路由跳转。

**完整流程图：**

```
用户访问任意 URL
        │
        ▼
  NProgress.start()  ← 显示顶部进度条
        │
        ▼
  有 Token 吗？（getToken()）
        │
   ┌────┴────┐
   │ 有      │ 没有
   ▼         ▼
 │ 是 /login 页面？        是白名单吗（/login, /register）？
 │   → 是：重定向到 /           → 是：放行
 │   → 否：继续                 → 否：重定向到 /login?redirect=原地址
 │
 ▼
 │ 是白名单？→ 放行
 │
 ▼
 │ 是锁定状态？
 │   → 当前不是 /lock 页面：跳到 /lock
 │   → 当前是 /lock 页面：跳到首页
 │
 ▼
 │ store.getters.roles 有值吗？（已拉取过用户信息？）
 │
 ├── 有：直接放行
 │
 └── 没有：
       isRelogin.show = true（防止重复弹窗）
       │
       ▼
       store.dispatch('GetInfo')  ← 调用 /getInfo，获取用户、角色、权限
       │
       ├── 失败 → LogOut → 跳到 /
       │
       └── 成功：
             store.dispatch('GenerateRoutes')  ← 调用 /getRouters，获取菜单树
             │
             ▼
             filterAsyncRouter()  ← 将后端返回的字符串路径转为 Vue 组件
             │
             ▼
             filterDynamicRoutes() ← 过滤前端定义的动态路由（按权限匹配）
             │
             ▼
             router.addRoutes()   ← 动态添加路由
             │
             ▼
             next({ ...to, replace: true }) ← 重新跳转

router.afterEach → NProgress.done()  ← 隐藏进度条
```

**关键概念：**

| 概念 | 说明 |
|------|------|
| **白名单** | `['/login', '/register']`，无需登录即可访问 |
| **Token** | 存在 Cookie `Admin-Token` 中，由 `js-cookie` 管理 |
| **isRelogin** | 导出对象 `{show: false}`，防止 401 时反复弹窗 |
| **roles** | 用户登录后从 `/getInfo` 获取，存在 `store.state.user.roles`。为空数组表示未登录或未拉取信息 |

#### 3.6.3 dynamicRoutes — 动态路由（按权限加载）

| `/monitor/job-log/index/:jobId` | `monitor:job:list` | 调度日志 |
| `/tool/gen-edit/index/:tableId` | `tool:gen:edit` | 修改生成配置 |

#### 3.6.4 路由元信息 meta

```javascript
meta: {
  title: '首页',        // 侧边栏和面包屑显示的名称

  affix: true,         // 固定页签（不可关闭）

  noCache: true,       // 不缓存页面
  link: 'http://...'   // 外链（iframe 展示）
}
```

#### 3.6.5 防重复跳转错误处理

```javascript
Router.prototype.push = function push(location) {
  return routerPush.call(this, location).catch(err => err)
}
```

覆盖了原生的 `push` 和 `replace` 方法，捕获重复导航导致的 `NavigationDuplicated` 错误。

### 3.7 `store/` — Vuex 状态管理

```
store/
├── index.js         ← 创建 Store 实例，注册所有模块

└── modules/

    ├── app.js       ← 侧边栏、设备
    ├── settings.js  ← 布局配置
    ├── tagsView.js  ← 页签管理
    ├── dict.js      ← 字典缓存
    └── lock.js      ← 屏幕锁定
```

#### 3.7.1 `store/modules/user.js` — 用户模块

**State:**

| 字段 | 初始值 | 来源 |
|------|--------|------|
| `token` | `getToken()`（从 Cookie 读取） | 登录后写入 |
| `id` | `''` | `/getInfo` 返回 `user.userId` |
| `name` | `''` | `/getInfo` 返回 `user.userName` |
| `nickName` | `''` | `/getInfo` 返回 `user.nickName` |
| `avatar` | `''` | `/getInfo` 返回（处理成完整路径） |
| `roles` | `[]` | `/getInfo` 返回的角色数组 |
| `permissions` | `[]` | `/getInfo` 返回的权限字符串数组 |

**Actions:**

| Action | 触发时机 | 流程 |
|--------|----------|------|
| `Login` | 用户点击登录 | 调用 `api/login.js` → 拿到 token → 存 Cookie → 更新 state → 解锁屏幕 |
| `GetInfo` | 路由守卫检查 roles 为空时 | 调用 `/getInfo` → 解构 user/roles/permissions → 更新 state。同时检查初始密码/过期密码 |
| `LogOut` | 用户点击退出 / 401 过期 | 调后端 `/logout` → 清空 state → 删 Cookie |
| `FedLogOut` | 前端主动登出（不调后端） | 直接清空 Token 和 Cookie |

#### 3.7.2 `store/modules/permission.js` — 权限+路由模块（核心）

**State:**

| 字段 | 说明 |
|------|------|
| `routes` | 完整路由表（constantRoutes + 动态生成的） |
| `addRoutes` | 动态添加的路由 |
| `defaultRoutes` | 默认路由（仅后端返回的菜单树） |
| `topbarRouters` | 顶部菜单栏路由 |
| `sidebarRouters` | 侧边栏菜单路由 |

**核心 Action: `GenerateRoutes`**

```
1. 调用 api/menu.js 的 getRouters() → 获取后端菜单树 JSON
2. 深拷贝两份 sdata（侧边栏用）和 rdata（完整路由用）
3. filterAsyncRouter(sdata, false, false)：
   - 将后端返回的 component 字符串（如 "system/user/index"）
   - 通过 loadView() 转为 Vue 组件
   - "Layout" → @/layout/index
   - "ParentView" → @/components/ParentView
   - "InnerLink" → @/layout/components/InnerLink
   - 其他 → require(`@/views/${view}`) 或 import(`@/views/${view}`)
4. filterAsyncRouter(rdata, false, true)：
   - 同上，但额外处理 children 路径拼接
5. filterDynamicRoutes(dynamicRoutes)：
   - 检查前端定义的动态路由所需的 permissions/roles
   - 有权限的加入 asyncRoutes
6. router.addRoutes(asyncRoutes) → 添加动态路由
7. commit 更新 state
```

**loadView 函数：**

```javascript
// 开发环境：使用 require（同步加载，提升热更新速度）
(resolve) => require([`@/views/${view}`], resolve)

// 生产环境：使用 import()（异步懒加载，提升首屏速度）
() => import(`@/views/${view}`)
```

#### 3.9.1 `request.js` — Axios 封装（核心文件）


**创建配置：**

**请求拦截器（request interceptor）：**

```
1. 检查是否需要 Token（config.headers.isToken !== false）
   → 需要：自动添加 Authorization: Bearer <token>

2. GET 请求参数处理：
   → 将 params 对象转换为 URL query string（使用 tansParams）

3. 防重复提交（Post/Put 请求）：
   → 比较当前请求的 url + data + 时间 与 sessionStorage 中上一次请求
   → 如果 url 和 data 相同且间隔 < interval（默认1000ms）→ 拒绝提交
```

| `601` | Message 警告提示 |
| 其他 | Notification 错误通知 |
| 网络错误 | "后端接口连接异常" |
| 超时 | "系统接口请求超时" |

| `$auth.hasPermi('system:user:list')` | 验证是否具备某权限 |

## 四、完整启动流程

```
1. 用户打开浏览器 → 加载 public/index.html
   ├── 显示加载动画（旋转圆环）
   └── 加载 JS bundle

2. main.js 执行
   ├── 注册 Element UI（设置全局 size 为 Cookie 中的值，默认 medium）
   ├── 注册全局组件和方法（this.getDicts, this.parseTime 等）
   ├── 注册插件（this.$tab, this.$auth, this.$cache, this.$modal, this.$download）
   ├── 注册指令（v-hasPermi, v-hasRole, v-clipboard 等）
   ├── 导入 SVG 图标
   ├── 导入 permission.js（启动路由守卫）
   └── 创建 Vue 实例：new Vue({ el: '#app', router, store, render: h => h(App) })

3. Vue 实例化 → App.vue 渲染 → <router-view> 根据当前 URL 渲染

4. 路由守卫（router.beforeEach）触发
   ├── 没有 Token：
   │   ├── URL 是 /login → 放行（显示登录页）
   │   ├── URL 是 /register → 放行（显示注册页）
   │   └── 其他 → 重定向到 /login?redirect=原URL
   │
   └── 有 Token：
       ├── URL 是 /login → 重定向到 /（已经登录了）
       ├── 已锁定且 URL 不是 /lock → 重定向到 /lock
       ├── roles 为空（未拉取用户信息）→ 三步曲：
       │   1. store.dispatch('GetInfo') → 调 /getInfo → 拿到 user/roles/permissions
       │   2. store.dispatch('GenerateRoutes') → 调 /getRouters → 拿到菜单树
       │      → filterAsyncRouter 将字符串转为 Vue 组件 → 生成完整路由
       │      → router.addRoutes() 添加动态路由
       │   3. next({ ...to, replace: true }) → 重新跳转
       └── roles 不为空 → 直接放行

5. 路由匹配到 Layout → 渲染布局框架
   ├── Sidebar（侧边栏菜单）
   ├── Navbar（顶部导航）
   ├── TagsView（页签栏）
   └── AppMain → <keep-alive> → <router-view> → 业务页面组件
```

## 五、请求流程详解

```
页面组件调用 API
        │
        ▼
api/system/user.js → listUser(query)
        │
        ▼
utils/request.js → service.get('/system/user/list', { params: query })
        │
        ▼
【请求拦截器】
  1. 检查是否需要 Token → 添加 Authorization: Bearer xxx
  2. GET 请求 → 将 params 序列化为 URL query string
  3. POST/PUT 请求 → 防重复提交检查（比较 sessionStorage 中的上次请求）
        │
        ▼
发送 HTTP 请求到 /dev-api/system/user/list
        │
        ▼
Vue CLI devServer proxy 拦截
  → 匹配 /dev-api 前缀
  → 转发到 http://localhost:8080/system/user/list
  → 去掉 /dev-api 前缀
        │
        ▼
后端 Spring Boot 处理 → 返回 JSON
        │
        ▼
【响应拦截器】
  1. 解析 res.data.code
  2. code === 200 → 返回 res.data
  3. code === 401 → 弹出重登录确认框
  4. code === 500 → Message 错误提示
  5. 其他 → Notification 错误通知
        │
        ▼
页面组件拿到数据 → 更新视图
```

## 六、动态路由加载流程图解

```
后端 GET /getRouters 返回的数据格式：
[
  {
    "name": "System",

    "component": "Layout",

[

       ↓
```

## 七、权限控制体系

权限控制有**三个层面**：

| 层面 | 实现方式 | 控制粒度 |
|------|----------|----------|
| **路由层** | `router.beforeEach` 守卫 + `getRouters` 动态路由 | 控制"能看到哪些菜单/页面" |
| **指令层** | `v-hasPermi` / `v-hasRole` 自定义指令 | 控制"能看到哪些按钮" |
| **函数层** | `$auth.hasPermi()` / `checkPermi()` 工具函数 | 控制"能执行哪些操作" |

### 使用示例

**1. 按钮权限（指令）：**
```html
<el-button v-hasPermi="['system:user:add']">新增</el-button>
<!-- 用户没有 system:user:add 权限时，该按钮自动从 DOM 中移除 -->
```

**2. 按钮权限（函数）：**
```javascript
if (this.$auth.hasPermi('system:user:edit')) {
  // 执行编辑逻辑
}
```

**3. 菜单权限（路由）：**
```javascript
// router/index.js
{
  path: '/system/user',
  permissions: ['system:user:list'],  // 只有拥有此权限才能访问
}
```

## 八、字典使用机制

RuoYi 的数据字典用于将数据库中存储的代码值（如 `0`、`1`）展示为用户可读的标签（如 `男`、`女`）。

### 字典工作流程：

```
1. 页面加载时调用 this.getDicts('sys_user_sex')
       │
2. dict 插件检查 store.state.dict.dict 缓存
       ├── 有缓存 → 直接返回
       └── 无缓存 → 调后端 /system/dict/data/type/sys_user_sex → 缓存到 store
       │
3. 返回字典数据 [{label: '男', value: '0'}, {label: '女', value: '1'}]
       │
4. 在模板中使用：
   - 表格列回显：DictTag 组件（彩色标签）
   - 表单下拉框：v-for 遍历字典数据生成 el-option
```

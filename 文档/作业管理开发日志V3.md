## V3 概述

V3 新增作业审核模块，实现"作业计划 → 作业审核 → 作业打卡"三段式业务流程。核心变更包括：

1. **作业审核模块**：新增 hw_review 表 + 完整后端/前端，支持审核通过/驳回
2. **计划创建权限管控**：仅施工方人员（非普通施工人员）或管理员可创建计划
3. **自动审核流转**：计划创建后自动生成审核记录
4. **5 状态生命周期**：hw_plan.status 从 4 状态扩展为 5 状态（0待审核 1待执行 2进行中 3已完成 4已取消）
5. **打卡准入控制**：仅审核通过的计划可进行打卡

## 四、各功能实现要点

### 4.1 作业审核模块（新增）

- **数据表**：`hw_review`，冗余 plan_name/work_type/construction_unit 字段便于审核列表展示
- **自动流转**：HwPlanServiceImpl.insertHwPlan() 中，计划插入后自动创建审核记录
- **审核通过**：HwReviewServiceImpl.approve() → 审核状态→已通过(1)，同步计划状态 0→1（待审核→待执行）
- **审核驳回**：HwReviewServiceImpl.reject() → 审核状态→已驳回(2)，同步计划状态 0→4（待审核→已取消）
- **重复审核拦截**：approve/reject 方法中校验 `!"0".equals(reviewStatus)`，防止重复操作

### 4.2 计划创建权限控制

- **校验时机**：HwPlanController.add() 方法中，在插入之前通过 validatePlanCreator() 校验
- **校验逻辑**：
  1. SecurityUtils.isAdmin() → 管理员直接放行
  2. 按当前用户名（SecurityUtils.getUsername()）查找 TbWorker（先按姓名查，再按手机号查）
  3. 未找到人员 → 提示"未找到您的人员信息"
  4. unit_type != '3' → 提示"仅施工方人员可创建作业计划"
  5. 查询 role_code 列表（selectRoleCodesByWorkerId）
  6. 仅含 'worker' 角色 → 提示"普通施工人员无权创建作业计划"

### 4.3 5 状态生命周期

| 状态值 | 状态名 | 触发方式 | 下一状态 |
|--------|--------|---------|---------|
| 0 | 待审核 | 计划创建 | 1（审核通过）/ 4（审核驳回/取消） |
| 1 | 待执行 | 审核通过 | 2（开始作业）/ 4（取消） |
| 2 | 进行中 | 首次进场打卡 | 3（标记完成）/ 4（取消） |
| 3 | 已完成 | 标记完成 | —（不可变更） |
| 4 | 已取消 | 审核驳回/取消 | 0（管理员恢复） |

### 4.4 打卡准入控制

- HwAttendanceServiceImpl.checkIn/checkOut 中，校验计划状态必须为 `"1"`（待执行）或 `"2"`（进行中）
- 不同状态给出明确错误提示：
  - status='0' → "该计划尚未通过审核，无法打卡"
  - status='3' → "该计划已完成，无法打卡"
  - status='4' → "该计划已取消，无法打卡"
- 首次进场打卡成功后，自动将计划状态从"1"（待执行）切换为"2"（进行中）

## 九、Bug 修复（2026-06-04 第二轮）

### Bug：审核弹窗"参与人员"显示为空

**现象**：审核页面点击"通过"或"驳回"后，弹窗中作业类型、施工单位、作业内容等字段正常显示，但"参与人员"一栏为空白（显示"—"）。

| 2 | `review/index.vue` | `this.reviewOpen = true` 写在 `Promise.all().then()` 回调**外面**，弹窗在 `getPlanWorkers` 数据返回之前就已打开。且直接使用已废弃的 `currentPlan.workers` 字段（varchar），未调用关联表接口 |

**修复内容**：

| 文件 | 改动 |
|------|------|
| `HwPlanController.java:73-85` | `return toAjax(hwPlanService.insertHwPlan(plan))` → `hwPlanService.insertHwPlan(plan); return success(plan.getPlanId())`。`useGeneratedKeys` 会在 insert 后回填 planId 到实体，直接返回真正的 ID |

- 仅 `HwPlanController.add` 的返回值格式变更（从 `int` 变为 `Long`），前端 `r.code === 200` 判断不变，完全兼容

## 十、功能改进 — 参与人员二级角色选择器（2026-06-04 第三轮）

### 改进背景

原参与人员选择器为平铺式 `el-select` 多选下拉，所有人员混在一起列出，名称后括号标注角色。随着人员增多，难以按角色筛选人员，用户体验差。

### 改进目标

改为**二级选择**：第一级展示角色列表（从人员管理模块的角色规则获取），点击角色后第二级展示该角色下的所有人员，勾选即可添加。

### 实现方案

**后端**：新增 `/worker/worker/rolesWithWorkers` 接口，返回按角色分组的人员数据。

| 文件 | 改动 |
|------|------|
| `TbWorkerRoleRelMapper.java` | 新增 `selectWorkersByRole()` 方法 |
| `TbWorkerRoleRelMapper.xml` | 新增 SQL：`SELECT o.role_code, o.role_name, w.id AS workerId, w.worker_name FROM tb_worker_role o JOIN tb_worker_role_rel rr ON o.id = rr.role_id JOIN tb_worker w ON w.id = rr.worker_id WHERE w.del_flag='0' AND w.status='0' AND o.status='0' ORDER BY o.id, w.id` |
| `TbWorkerController.java` | 新增 `GET /rolesWithWorkers` 端点（无 @PreAuthorize，与 /options、/allRoleNames 一致） |

**返回数据格式**：
```json
[
  { "roleCode": "applicant", "roleName": "作业申请人", "workerId": 2, "workerName": "李四" },
  { "roleCode": "guardian",  "roleName": "作业监护人", "workerId": 3, "workerName": "王五" },
  ...
]
```

前端接收后按 `roleCode` 分组渲染。

**前端**：`plan/index.vue` 参与人员区域从 `el-select` 改为 `el-popover` 二级面板。

| 文件 | 改动 |
|------|------|
| `api/worker/worker.js` | 新增 `getRolesWithWorkers()` 接口函数 |
| `plan/index.vue` — 模板 | 替换 `<el-select>` 为 `<el-popover>` 内嵌左右分栏：左栏角色列表、右栏 `el-checkbox-group` 人员勾选 |
| `plan/index.vue` — 脚本 | import 改为 `getRolesWithWorkers`；data 中 `workerOptions`/`roleNameMap` 替换为 `roleGroups`/`activeRole`/`checkedWorkerIds`/`workerPopVisible`；新增 `computed: { activeRoleWorkers, selectedWorkersText }`；`loadWorkerOptions` 改为调用新接口并分组；新增 `onWorkerCheckChange` 双向同步 `checkedWorkerIds` ↔ `selectedWorkers` |

**UI 交互**：

```
点击"点击选择参与人员"输入框
        ┌────────────────┬──────────────────┐
        │  作业申请人 (1)  │ ☑ 李四            │
        │▶ 作业批准人 (1)  │ ☐ 刘某某          │
        │  作业监护人 (1)  │                  │
        │  监理人员   (1)  │  勾选/取消勾选     │
        │  施工方项目经理  │                  │
        │  ...           │                  │

输入框显示：已选 3 人
标签栏：李四(作业申请人)  王五(作业监护人)  ...
```

**设计要点**：
- `checkedWorkerIds` 为全局选中集合，切换角色时勾选状态保持
- 标签按角色着色区分（显示角色中文名）

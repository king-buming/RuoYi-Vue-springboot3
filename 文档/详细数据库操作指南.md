# 若依项目数据库操作详细指南

> **适用前提**：你已安装 MySQL 8.x。如果还没装，先去 [MySQL 官网](https://dev.mysql.com/downloads/mysql/) 下载安装，记住安装时设置的 root 密码。

---

## 零、确认 MySQL 是否运行

在进行任何操作之前，先确认 MySQL 服务正在运行。

### 打开 Windows 服务管理器（4 种方式，任选一个即可）

| 方式 | 操作 |
|------|------|
| 方式 A | 按 `Win + R`，输入 `services.msc`，回车 |
| 方式 B | 按 `Win` 键，搜索"服务"，点击打开 |
| 方式 C | 任务管理器 → "服务"标签页 |
| 方式 D | `Ctrl + Shift + Esc` → "服务"标签页 |

### 检查 MySQL 状态

在服务列表中找到 **MySQL80**（或类似名称，根据你安装的版本不同，可能叫 `MySQL`、`MySQL57`、`MySQL8.0` 等）：

- 如果"状态"列显示 **"正在运行"** → 正常，继续下一步
- 如果"状态"列为空 → **右键 → 启动**

> 如果找不到 MySQL 服务，说明 MySQL 还没有安装，请先安装 MySQL 8.x。

---

## 一、在 IDEA 中连接 MySQL

**你不需要安装 Navicat。IDEA 自带的 Database 工具完全可以替代。**

### 1.1 打开 Database 面板

IDEA 右侧边栏，找到 **Database** 图标（一个圆柱体形状的图标），点击打开。

![图示：IDEA 右侧边栏，Database 图标是一个圆柱体](示意图：右侧边栏圆柱体图标)

如果右侧边栏没有这个图标，有两种打开方式：

- **快捷键**：按 `Alt + 8`（推荐）
- **菜单路径**：顶部菜单栏 → `View` → `Tool Windows` → `Database`

### 1.2 添加 MySQL 数据源

1. 在 Database 面板中，点击左上角的 **`+` 号**
2. 在弹出的菜单中，选择 **Data Source**
3. 在子菜单中选择 **MySQL**

> 如果子菜单中没有 MySQL，可以输入"MySQL"搜索

### 1.3 填写连接信息

会弹出一个"Data Sources and Drivers"对话框，在 **General** 标签页填写以下内容：

#### "General" 标签页的各个输入框

| 字段 | 填什么 | 说明 |
|------|--------|------|
| **Host** | `localhost` | 本机就是 localhost |
| **Port** | `3306` | MySQL 默认端口，安装时没改过就是 3306 |
| **User** | `root` | 你安装 MySQL 时设置的用户名，默认是 root |
| **Password** | `你的密码` | 你安装 MySQL 时设置的密码 |
| **Database** | **先留空** | 我们还没创建数据库，后面再填 |

> **注意**：如果"Database"字段填了不存在的数据库名，下面测试连接时会报错。所以这个字段先空着。

#### 填写完成后的效果示意

```
┌─────────────────────────────────────────┐
│  Host:     localhost                     │
│  Port:     3306                          │
│  User:     root                          │
│  Password: ••••••••                      │
│  Database: (空着)                         │
└─────────────────────────────────────────┘
```

### 1.4 测试连接（关键步骤，用来确认信息都填对了）

在对话框左下角，点击 **Test Connection** 按钮。

- 如果有**绿色的 ✓** 提示 `Successful` → 连接成功，点击 **OK** 保存
- 如果报错，对号入座检查：

| 常见错误提示关键词 | 可能的原因 | 怎么办 |
|-------------------|-----------|--------|
| `Communications link failure` | MySQL 服务没有启动 | 回到"第零步"启动 MySQL 服务 |
| `Access denied for user 'root'` | 用户名或密码填错了 | 重新输入正确的密码 |
| `Unknown database` | Database 字段填了不存在的数据库名 | 把 Database 字段清空 |
| `Cannot connect to localhost:3306` | 端口 3306 被防火墙拦截，或端口号不对 | 确认 MySQL 端口，或检查防火墙 |

### 1.5 保存连接

点击 **OK**，IDEA 右侧的 Database 面板中就会出现你刚添加的数据源。

此时你看到的数据源下面不会有任何数据库（或者只有系统自带的几个数据库如 `information_schema`、`mysql` 等），这很正常——我们接下来创建 `ry-vue` 数据库。

## 二、创建空数据库 `ry-vue`

### 2.1 方式一：右键菜单创建（推荐，不用写 SQL）

1. 在 Database 面板中找到你刚才连接的数据源（名字默认是 `root@localhost`）
2. **右键点击数据源**（注意是右键数据源本身，不是下面的子项）
3. 在弹出的菜单中：**New** → **Database**
4. 在弹出的对话框中：
   - **Name**（数据库名）：输入 `ry-vue`
   - **Character set**（字符集）：`utf8`（或 `utf8mb4`，两者都可以，推荐选 `utf8mb4` 因为支持 emoji）
   - **Collation**（排序规则）：可以留空，系统会自动匹配
5. 点击 **OK**

### 2.2 方式二：用 SQL 语句创建（适合想练习 SQL 的情况）

1. 在 Database 面板中，**右键点击数据源** → **Open Query Console**（或按 `Ctrl + Shift + F10`）
2. 在打开的编辑窗口中输入：

```sql
CREATE DATABASE `ry-vue` DEFAULT CHARACTER SET utf8 COLLATE utf8_general_ci;
```

3. 按 `Ctrl + Enter` 执行这行 SQL
4. 执行成功后，Database 面板会刷新，你会看到 `ry-vue` 出现在数据源下面

### 2.3 验证数据库创建成功

在 Database 面板中，展开数据源，你应该能看到 `ry-vue` 数据库了。此时数据库是空的（schema 为空，里面 0 张表），这是正常的，接下来我们要导入表。

## 三、导入 SQL 文件

项目 `sql/` 目录下有两个 SQL 文件，**必须按顺序导入**：

| 顺序 | 文件名 | 内容说明 |
|------|--------|---------|
| 第 1 个 | `ry_20260417.sql` | 系统核心表 + 初始化数据（用户、角色、菜单、部门、字典、参数、日志等，约 40 张表） |
| 第 2 个 | `quartz.sql` | Quartz 定时任务相关表（约 11 张表） |

### 3.1 导入第一个文件：`ry_20260417.sql`

1. 在 Database 面板中，找到并**右键点击 `ry-vue` 数据库**（注意是右键数据库，不是数据源）
2. 在弹出的菜单中：**Run SQL Script...**
3. IDEA 会弹出文件选择窗口，导航到项目目录的 `sql/` 文件夹：
   ```
   你的项目目录\sql\ry_20260417.sql
   ```
   选中这个文件，点击 **Open**
4. 接下来会弹出"Execute SQL Script"对话框，确认以下设置：

   | 设置项 | 应该是什么 | 说明 |
   |--------|-----------|------|
   | Data Source | `root@localhost` | 你刚才配置的 MySQL 连接 |
   | Database | `ry-vue` | 目标数据库 |
   | Charset | `UTF-8` | 编码格式 |

   这些默认就是正确的，直接点 **Run** 即可。

5. 等待执行完成。IDEA 底部的"Run"面板会显示执行日志，正常结束时类似：

   ```
   1 row(s) affected
   ...
   Execution finished with no errors.
   ```

   **如果时间很长卡住了**，检查你的网络代理设置（项目使用了 `127.0.0.1:7897` 代理），数据源不需要代理，确认 SQL 执行窗口中没有报错即可。

### 3.2 导入第二个文件：`quartz.sql`

重复 3.1 的步骤，这次选择 `sql/quartz.sql`：

1. 右键点击 `ry-vue` 数据库
2. **Run SQL Script...**
3. 选择 `sql/quartz.sql`
4. 点击 **Run**

### 3.3 验证导入是否成功

1. 在 Database 面板中，展开 `ry-vue` 数据库 → 展开 **tables**（表）
2. 你应该能看到几十张表，主要的表包括：

   | 表名前缀 | 表数量 | 说明 |
   |----------|--------|------|
   | `sys_` 开头 | ~20 张 | 系统核心表（user, role, menu, dept, config, dict 等） |
   | `qrtz_` 开头 | ~11 张 | Quartz 定时任务表 |
   | 其他 | 几张 | 比如 `gen_table`（代码生成）、`sys_job_log` 等 |

   如果展开能看到几十张表，说明 SQL 导入成功了。

### 3.4 关于 SQL 文件的说明（理解你在干什么）

`ry_20260417.sql` 这个文件里，**每条建表语句之前都有 `DROP TABLE IF EXISTS`**：

```sql
-- 文件中的逻辑结构（示例）
drop table if exists sys_dept;       -- 先删旧表（如果存在的话）
create table sys_dept (...);         -- 再创建新表
insert into sys_dept values (...);   -- 再插入初始数据

drop table if exists sys_user;
create table sys_user (...);
insert into sys_user values (...);
-- ... 以此类推
```

这意味着：**如果你导错了一次，再次导入同一个文件不会出错**——它会把旧表删掉重建。

## 四、修改项目中的数据库配置

导入完成后，还需要让项目的配置文件知道你的 MySQL 密码。

### 4.1 打开配置文件

在 IDEA 的项目文件树中，找到并打开：

```
ruoyi-admin → src → main → resources → application-druid.yml
```

完整路径：
```
ruoyi-admin/src/main/resources/application-druid.yml
```

### 4.2 修改用户名和密码

找到文件的第 10-11 行：

```yaml
master:
    url: jdbc:mysql://localhost:3306/ry-vue?useUnicode=true&characterEncoding=utf8&zeroDateTimeBehavior=convertToNull&useSSL=true&serverTimezone=GMT%2B8
    username: root          # ← 改这里
    password: password      # ← 改这里
```

把 `username` 和 `password` 后面的值，改成你自己的 MySQL 用户名和密码：

```yaml
master:
    url: jdbc:mysql://localhost:3306/ry-vue?useUnicode=true&characterEncoding=utf8&zeroDateTimeBehavior=convertToNull&useSSL=true&serverTimezone=GMT%2B8
    username: root          # 如果用户名就是 root，这行不用改
    password: 你的真实密码    # 把 password 改成你在 IDEA 连接 MySQL 时输入的密码
```

> 注意：如果你的用户名本来就是 `root`，只需要改 `password` 那一行就行。

### 4.3 保存文件

按 `Ctrl + S` 保存。

## 五、检查清单（一步一步确认）

在启动项目之前，对着下面这 5 条逐一确认：

| 序号 | 检查项 | 怎么确认 |
|------|--------|---------|
| 1 | MySQL 服务正在运行 | 服务管理器（`services.msc`）中 MySQL 状态 = "正在运行" |
| 2 | IDEA 能连上 MySQL | Database 面板中数据源显示为正常状态（非红色） |
| 3 | `ry-vue` 数据库已创建 | Database 面板展开能看到 `ry-vue` 数据库 |
| 4 | 表已经导入 | 展开 `ry-vue` → tables，能看到几十张 `sys_` 和 `qrtz_` 开头的表 |
| 5 | 配置文件密码已修改 | `application-druid.yml` 的 password 是你 MySQL 的真实密码 |

5 条全部确认无误后，就可以启动项目了。

## 六、常见问题

### Q1：IDEA 的 Database 面板在哪里？我找不到。

**A**：三种方法：

1. 按快捷键 `Alt + 8`
2. 菜单栏：`View` → `Tool Windows` → `Database`
3. 如果上面两种都找不到，可能是因为你用的是 IDEA Community Edition（社区版）。社区版默认没有完整的 Database 工具，但可以通过插件安装：
   - `File` → `Settings` → `Plugins`
   - 搜索 `Database Navigator` 插件，安装后重启 IDEA
   - 或者：考虑用命令行导入 SQL（见 Q2）

### Q2：命令行怎么导入 SQL？（IDEA 导入不成功的备选方案）

**A**：打开 Windows 终端（PowerShell 或 CMD），执行以下命令：

```bash
# 先进入项目根目录（包含 sql/ 文件夹的目录）
cd C:\Users\ASUS\Desktop\RuoYi-Vue-springboot3-main\RuoYi-Vue-springboot3

# 创建数据库（如果还没创建的话）
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS `ry-vue` DEFAULT CHARACTER SET utf8 COLLATE utf8_general_ci;"

# 导入第一个文件
mysql -u root -p ry-vue < sql\ry_20260417.sql

# 导入第二个文件
mysql -u root -p ry-vue < sql\quartz.sql
```

> 注意：`-u root` 中的 `root` 是你的 MySQL 用户名，`-p` 执行后会提示你输入密码。如果你的用户名不是 root，把 `root` 改成你的用户名。

### Q3：导入 SQL 时 IDEA 报"no database selected"？

**A**：说明你没有在 `ry-vue` 数据库上执行导入。回到第三步，确认你是**右键点击 `ry-vue` 数据库**来导入的，而不是右键数据源，也不是在 Query Console 里执行的。

### Q4：新建数据库时找不到 utf8 字符集选项？

**A**：下拉列表中通常显示为 `utf8` 或 `utf8mb4`：
- `utf8mb4`：支持完整的 Unicode，包括 emoji 表情（推荐）
- `utf8`：MySQL 的 `utf8` 实际上是 `utf8mb3`，大部分中文场景够用

两个都可以，如果要完全兼容，选 `utf8mb4`。

### Q5：我忘了 MySQL 的 root 密码怎么办？

**A**：Windows 下 MySQL 8.x 忘记密码的修复方法：

1. 停止 MySQL 服务（`services.msc` 中右键停止）
2. 以管理员身份打开终端，进入 MySQL 安装目录的 `bin` 文件夹
3. 执行 `mysqld --console --skip-grant-tables --shared-memory` 启动 MySQL（跳过权限验证）
4. 另开一个终端，执行 `mysql -u root`（不需要密码即可登入）
5. 执行 `ALTER USER 'root'@'localhost' IDENTIFIED BY '你的新密码';`
6. 关闭两个终端，重启 MySQL 服务

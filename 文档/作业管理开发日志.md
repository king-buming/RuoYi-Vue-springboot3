#### HwAttendanceMapper（含专用业务查询）

| 方法 | SQL 逻辑 | 用途 |
|------|---------|------|
| `selectHwAttendanceList` | 多条件筛选：plan_id、user_name LIKE、check_type、check_method、check_status、check_time 范围 | 打卡记录列表 |
| `selectHwAttendanceById` | 主键查询 | 记录详情 |
| `insertHwAttendance` | 标准插入 | 写入打卡记录 |
| `selectLastCheckIn(planId, userId)` | `WHERE plan_id=? AND user_id=? AND check_type='0' ORDER BY check_time DESC LIMIT 1` | **进场重复检查**：判断该人员是否已打过进场卡 |
| `selectOvertimeUncheckedPlans(minutes)` | 关联 hw_plan，`plan_work_time < DATE_SUB(NOW(), INTERVAL #{minutes} MINUTE) AND NOT EXISTS (进场打卡)` | **进场超时检测**：30分钟内未进场打卡的计划 |
| `selectOvertimeCheckIns(minutes)` | `check_type='0' AND check_time < DATE_SUB(NOW(), INTERVAL #{minutes} MINUTE) AND NOT EXISTS (离场打卡)` | **离场超时检测**：进场超8小时未离场打卡的记录 |

### 2.3 业务逻辑层（Service）—— 6 个文件（3 接口 + 3 实现）

#### IHwPlanService / HwPlanServiceImpl

标准 CRUD，insert 前调用 `SecurityUtils.getUsername()` 设置 `createBy`，update 前设置 `updateBy`。

#### IHwWorkerService / HwWorkerServiceImpl

标准 CRUD，模式同 HwPlan。

#### IHwAttendanceService / HwAttendanceServiceImpl — 核心业务逻辑

| 方法 | 业务规则 |
|------|---------|
| `checkIn(HwAttendance)` | ① 校验 `hw_plan` 存在且 `status='0'` → ② 调 `selectLastCheckIn`，若已存在进场记录则抛 `ServiceException("该人员已进场打卡，请先完成离场打卡")` → ③ 设置 `checkType="0"`, `checkTime=new Date()`, `checkStatus="0"` → ④ 写入数据库 |
| `checkOut(HwAttendance)` | ① 校验 `hw_plan` 存在 → ② 调 `selectLastCheckIn`，若为 null 则抛 `ServiceException("未找到进场打卡记录，请先进场打卡")` → ③ 校验 `checkOut.checkTime > checkIn.checkTime` → ④ 设置 `checkType="1"`, `checkStatus="0"` → ⑤ 写入数据库 |

## 七、2026-06-05 证件到期提醒

### 7.1 修改背景

手机端消息中心需要增加证件到期提醒能力：人员证件距离到期 7 天内时，自动生成未读通知。

### 7.2 修改文件

| 文件 | 修改内容 |
|------|----------|
| `ruoyi-quartz/src/main/java/com/ruoyi/quartz/task/CertExpiryJob.java` | 将提醒窗口从 30 天调整为 7 天；使用自然日期计算剩余天数；过期证件自动标记 `auditStatus=3`；新增通知防重复逻辑 |
| `ruoyi-system/src/main/java/com/ruoyi/system/mapper/TbNotificationMapper.java` | 新增 `countByWorkerTypeBiz()`，用于判断同一业务通知是否已存在 |
| `ruoyi-system/src/main/resources/mapper/system/TbNotificationMapper.xml` | 新增 `countByWorkerTypeBiz` SQL |
| `sql/updates/20260605_cert_expiry_job.sql` | 更新定时任务备注为“7天内到期提醒，过期自动标记” |

### 7.3 当前规则

- 每天凌晨 2 点执行定时任务：`certExpiryJob.execute`。
- 证件 `expireDate` 距离当天 `0~7` 天时，写入一条 `cert_warn` 通知。
- 证件已经过期时，写入一条 `cert_expire` 通知，并把证件状态更新为 `auditStatus=3`。
- 同一个人员、同一个证件、同一种通知类型只写入一次，避免每天重复刷消息。

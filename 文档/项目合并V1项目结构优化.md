# 合并V1项目结构优化日志

> 优化日期：2026-06-02  
> 优化目标：将 `com.ruoyi.worker.*` 独立包结构合并到 `com.ruoyi.system.*` 标准分层结构中

---

## 一、优化背景

从 ruoyiV1 合并的人员管理模块（worker）使用了独立的包结构 `com.ruoyi.worker.*`，而作业管理模块（homework）已经按照 RuoYi 标准分层放在 `com.ruoyi.system.*` 下。两个同级模块的包结构不一致：

| 子模块 | 优化前包路径 | 问题 |
|--------|------------|------|
| 作业管理 (homework) | `com.ruoyi.system.domain/mapper/service` + `com.ruoyi.web.controller.homework` | 符合标准 |
| 人员管理 (worker) | `com.ruoyi.worker.domain/mapper/service` + `com.ruoyi.worker.controller` | 独立包，结构重复 |

---

## 四、无需变更的配置

以下配置无需修改，已验证：

### 4.1 MyBatis Mapper 扫描
`application.yml` 中配置为 `classpath*:mapper/**/*Mapper.xml`，使用通配符自动发现所有路径下的 XML 文件，移动后无需修改。

### 4.2 Spring 组件扫描
`@SpringBootApplication` 默认扫描 `com.ruoyi` 包及其子包，新老路径都在扫描范围内。

### 4.3 前端 API 路径（Vue 管理后台）
前端 `ruoyi-ui/src/api/worker/*.js` 使用 URL 路径（如 `/worker/worker/list`），与 Controller 的 `@RequestMapping` 匹配。Java 包路径变更不影响 URL 映射，无需修改。

涉及 6 个前端 API 文件：`worker.js`, `audit.js`, `cert.js`, `checkin.js`, `face.js`, `role.js`

### 4.4 前端 Vue 视图
6 个 Vue 页面文件（`src/views/worker/*/index.vue`）通过 API 模块调用后端，无需修改。

### 4.5 RuoYi-App 移动端
App 端使用 `/app/auth/*`、`/app/checkin/*`、`/app/worker/*` 接口，App 控制器 `@RequestMapping` 路径未变，无需修改。

涉及 3 个页面：`pages/worker/login.vue`, `pages/worker/checkin.vue`, `pages/worker/records.vue`

### 4.6 数据库/SQL
SQL 文件（`ry_20260417.sql`）中的建表语句和菜单 INSERT 与 Java 包路径无关，无需修改。

### 4.7 SecurityConfig
`SecurityConfig.java` 中的 `/app/**` 路径放行已在之前的合并中正确配置，无需修改。

### 4.8 Quartz 定时任务
`HwAttendanceTimeoutJob` 使用 `com.ruoyi.common.*` 和 `com.ruoyi.system.*`，不涉及 worker 包，无需修改。

## 八、后续部署注意事项

1. **重启后端服务**：本次优化仅涉及 Java 包路径变更，URL 映射和数据库结构完全不变，重启后端即可生效。
2. **前端无需重新构建**：API 路径未变，前端代码零改动。
3. **App 端无需重新打包**：App 控制器接口路径未变。
4. **数据库无需重新导入**：SQL 文件与 Java 包无关。
5. **IDE 清理**：如果 IDEA 报红，执行 `File → Invalidate Caches` 或 `mvn clean install` 刷新。

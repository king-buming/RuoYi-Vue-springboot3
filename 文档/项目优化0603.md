# 项目优化记录 2026-06-03

## 优化目标

去除作业管理模块（homework）中重复的人员管理子模块（HwWorker），统一使用外部同等级的人员信息管理模块（TbWorker 系列表）。

## 背景

项目中有两套人员管理体系：

| 模块 | 表前缀 | 位置 | 说明 |
|------|--------|------|------|
| 作业管理-人员管理 | `hw_worker` | 作业管理子菜单 | 9角色体系，单表设计 |
| 人员信息管理 | `tb_worker` 系列（7张表） | 顶级独立模块 | 6大业务体系（档案/角色/资质/人脸/审核/打卡） |

作业管理模块的"人员管理"子模块功能已被外部独立的人员信息管理模块完整覆盖，属于重复建设。本次优化将作业管理模块中的 HwWorker 子模块完全移除，所有人员数据统一从外部 TbWorker 模块获取。

## 二、修改的文件

### 2.1 后端 — HwAttendanceServiceImpl.java

**文件路径**：`ruoyi-system/src/main/java/com/ruoyi/system/service/impl/HwAttendanceServiceImpl.java`

**改动内容**：

1. **替换依赖注入**：`HwWorkerMapper` → `TbWorkerMapper` + `TbWorkerFaceMapper`

```java
// 旧
@Autowired private HwWorkerMapper hwWorkerMapper;

// 新
@Autowired private TbWorkerMapper tbWorkerMapper;
@Autowired private TbWorkerFaceMapper tbWorkerFaceMapper;
```

2. **修改 `doCheckInValidation()` 人脸校验逻辑**：由于 TbWorker 人脸数据存储在独立的 `tb_worker_face` 表中，不再直接从 worker 实体获取 faceImage，改为两步查询：

```java
// 旧：直接从 hw_worker 表获取
HwWorker worker = hwWorkerMapper.selectHwWorkerByUserId(userId);
String faceImage = worker.getFaceImage();

// 新：先查人员档案，再查人脸底图
TbWorker worker = tbWorkerMapper.selectTbWorkerById(userId);
TbWorkerFace query = new TbWorkerFace();
query.setWorkerId(userId);
List<TbWorkerFace> faces = tbWorkerFaceMapper.selectTbWorkerFaceList(query);
String faceImgUrl = faces.get(0).getFaceImgUrl();
```

3. **调整错误提示**：人员不存在时提示"未找到该人员信息"（原"未找到该人员的人脸底图"合并为两步分别提示）。

### 2.2 前端 — 作业计划页面

**文件路径**：`ruoyi-ui/src/views/homework/plan/index.vue`

**改动内容**：

1. **API 导入变更**：从作业管理自带的 worker API 改为外部人员管理模块 API

```javascript
// 旧
import { listWorker } from "@/api/homework/worker"

// 新
import { listWorker, getAllRoleNames } from "@/api/worker/worker"
```

2. **人员选项加载逻辑重构**：

```javascript
// 旧：单次查询，直接使用 hw_worker 的 roleType 字段
listWorker({ pageNum: 1, pageSize: 999, status: '0' }).then(r => { this.workerOptions = r.rows })

// 新：并行查询人员列表 + 角色映射
Promise.all([
  listWorker({ pageNum: 1, pageSize: 999, status: '0' }),
  getAllRoleNames()
]).then(([workerRes, roleRes]) => {
  // 构建 workerId → roleName 映射
  const roleMap = {}
  roleRes.data.forEach(r => { roleMap[r.workerId] = r.roleName })
  // 合并数据
  this.workerOptions = workerRes.rows.map(w => ({
    ...w,
    workerId: w.id  // TbWorker 主键为 id，统一转为 workerId
  }))
})
```

3. **角色显示适配**：TbWorker 使用 M:N 角色关联（`tb_worker_role_rel`），不再有单一 `roleType` 字段。改为通过 `getAllRoleNames()` 接口获取角色名称映射显示。

4. **字段映射**：`w.workerId` → `w.id`（TbWorker 主键名不同）。

## 四、数据关系变更

```
旧架构：
hw_plan ──┬── hw_attendance（user_id → hw_worker.worker_id）
           ├── hw_plan_worker（worker_id → hw_worker.worker_id）
           └── hw_worker（自有人员表）

新架构：
hw_plan ──┬── hw_attendance（user_id → tb_worker.id）
           └── hw_plan_worker（worker_id → tb_worker.id）
tb_worker ──┬── tb_worker_cert
             ├── tb_worker_role_rel ── tb_worker_role
             ├── tb_worker_face
             ├── tb_worker_audit
             └── tb_worker_checkin
```
